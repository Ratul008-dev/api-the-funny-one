Why API is the funny one ?

The meaning of API is Application Programming Interface. It’s mainly a frontend-based concept that helps us display data or other elements in a convenient and readable way. For example: If we open any shopping app, we see a list of products with their images and prices. If we open a music app, we see songs, artists, and playlists — all beautifully displayed. The language used by most APIs is JSON (JavaScript Object Notation) — which, in its raw form, isn’t very readable for humans. So, we “parse” or convert this data and present it on our public website using the fetch() method in JavaScript. So… what’s the funny thing here? It’s the moment of success when we finally see our data appear correctly on the screen — when the syntax is clean, the code works, and everything flows just as we wanted. That happiness, that tiny victory — that’s the real funny part of APIs. So let’s dive in deep and enjoy the fun of fetching!

About

There are several types of APIs, but in this project, we’re focusing mainly on Public APIs and Private APIs. Public APIs are open and available for everyone to use — just like the Nobel Laureate API we’ve fetched here. They help developers learn, experiment, and build amazing things using real-world data. On the other hand, Private APIs are restricted — they’re mostly used inside an organization or app, where access is limited for security and privacy reasons. To parse and display these data beautifully on screen, we’ve used JavaScript methods like fetch() along with loops such as forEach(). We also used DOM manipulation — creating new elements dynamically using document.createElement(), connecting them with appendChild(), and targeting sections using document.getElementById().
In total, this project works with three APIs —
1️⃣ One Public API (Nobel Laureate Data)
2️⃣ Two Custom APIs (locally created JSON data)

1.What is a Custom Api

A custom API is a self-created Application Programming Interface designed to share and manage data or features specific to your own project. Unlike public APIs that provide predefined data from external sources, a custom API gives you complete control over the structure, content, and access of your data. It allows developers to experiment, learn, and build personalized solutions while understanding how requests, responses, and data parsing work in real-world applications — making it ideal for practical learning and tailored functionality.

Once the fetch() request succeeds, it gives a response object (res). But this isn’t raw data yet — it’s a Response stream. Calling .json() converts that stream into actual JSON data (JavaScript Object Notation) that we can easily work with. This also returns a Promise.

We loop through every item (each engine) in the JSON array. For each item, we create a new

What is a custom API with external link?

A custom API with an external link is a self-built data source where some information is stored separately outside the main code. Instead of keeping everything inside JavaScript or HTML, the API connects to external files or links such as JSON files, images, icons, documents, or even online resources. This approach keeps your project clean, organized, and easier to update. Developers often use external links to store large data, media, or reusable assets that can be fetched whenever needed. Custom APIs with external links create a flexible structure, improve performance, and make real-world applications more scalable and professional.

2.What is Public API ?

A public API is an Application Programming Interface that is openly available for anyone to use without special permissions. Companies and developers release public APIs so others can access their data or services, such as weather information, currency rates, movie details, or social media features. Public APIs usually come with documentation that explains endpoints, usage limits, and authentication requirements. Some public APIs are completely free, while others need an API key or have paid tiers. They allow developers to integrate external data into websites or apps easily, enabling faster development and richer user experiences.

Follow these steps to show this data on HTML page



  function displaydata() {
  tablebody.innerHTML = "";
  const nextset = data.slice(currentindex, currentindex + limit);
  nextset.forEach(laureate => {
    const firstPrize = laureate.nobelPrizes?.[0] || {};
    const row = document.createElement('tr');
    row.innerHTML = `
  ${laureate.fullName?.en || "N/A"}
      ${laureate.gender || "N/A"}
      ${firstPrize.awardYear || "N/A"}
      ${firstPrize.motivation?.en || "N/A"}
  `;
    tablebody.appendChild(row);
  });
  }

Elaboration of this code.



     fetch(apiurl)
     .then(res => res.json())
     .then(json => {
       data = json.laureates;
       displaydata();
     })

fetch(apiurl) sends a request to the Nobel Prize API.
.then(res => res.json()) converts the raw response into usable JSON.
json.laureates contains the actual list of laureates, so we store it inside data.
Finally, we call displaydata() to show the result on the page.



   tablebody.innerHTML = "";

This ensures the table becomes empty before inserting new rows.



   const nextset = data.slice(currentindex, currentindex + limit);

We do not display all laureates at once.
slice() picks only the next batch (example → 10 items).
This is the heart of your pagination logic.



   nextset.forEach(laureate => {

This will create one table row for each laureate.



   const firstPrize = laureate.nobelPrizes?.[0] || {};

?. prevents errors if nobelPrizes doesn’t exist.
If no prize exists, it returns {} so that fields become "N/A".



    const row = document.createElement('tr');

This makes an empty row where we will put our table data.



     row.innerHTML = `
     ${laureate.fullName?.en || "N/A"}
     ${laureate.gender || "N/A"}
     ${firstPrize.awardYear || "N/A"}
     ${firstPrize.motivation?.en || "N/A"}
  `;

Here we display: Full Name → laureate.fullName.en
Gender → laureate.gender
Award Year → firstPrize.awardYear
Motivation → firstPrize.motivation.en
All fields use || "N/A" to avoid blank values.



     tablebody.appendChild(row);
     });

This officially adds the row into the table on your webpage.
